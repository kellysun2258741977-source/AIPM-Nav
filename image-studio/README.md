# Image Studio · 图片生成工作流

> 一句粗略的画面需求，看它走完 **需求 → 提示词 → 出图** 的可见两道工序：先用文本模型把模糊需求扩写成专业图像提示词，再交给 **gpt-image-1** 出图、一次多张「抽选」、参考图改图。
> 同仓库 [AIPM-Nav](../) 下的独立子应用，与 [PRD Agent](../prd-agent/) 形成「单步工具 vs 多步自治」之外的另一组对照：**确定性工作流 vs 黑盒一键生成**。

## ✨ 它解决什么问题

直接把「我想要张科技感的产品 Banner」丢给图像模型，出图往往平庸——因为**提示词的质量决定了出图质量的上限**。这个工作流把「写好提示词」显式拆成第一道工序：

1. **① 需求 → 图像提示词**：文本模型（默认 `gpt-4o-mini`）把粗略需求扩写成结构完整的英文提示词（主体 / 场景 / 构图 / 光线 / 风格 / 画质），并给出中文回译与微调建议。**逐字流式**呈现思路。
2. **② 人在环中（HITL）**：出图前提示词可**直接编辑**，点微调建议一键追加。控制权交回给人——这正是「工作流」相对「黑盒一键生成」的价值。
3. **③ 提示词 → 出图**：交给图像模型（默认 `gpt-image-1`），一次生成 1~4 张供**抽选**，每张可放大 / 下载 / 「以此改图」。

此外支持：

- **参考图改图**：上传图片 + 修改诉求，由文本模型写编辑指令后调用 `images/edits`（gpt-image-1 支持多张参考图）。
- **本地历史**：每次出图把需求、提示词与一张缩略图存到浏览器 `localStorage`，可回看、点选复用。

## 🔌 两种运行模式

| 模式 | 是否需要 Key | 用途 |
| --- | --- | --- |
| **演示模式（默认）** | 否 | 零配置跑通完整工作流。文本模型用脚本化模板扩写，图像模型产出**渐变占位图**，方便随时展示链路。 |
| **BYOK · 真实模型** | 是 | 填入自己的 OpenAI Key，由**真实模型**驱动：文本模型写提示词、图像模型真实出图。兼容 OpenAI 协议端点。 |

两种模式共用**同一套**工作流编排与 UI——演示模式不是「假页面」，而是把真实模型替换成可预测的桩。

> 🔒 API Key 只保存在浏览器本地 `localStorage`，不会上传到任何服务器（本项目没有后端）。

## 🚀 运行方式

纯静态前端，**零依赖、零构建**。ES Module 需要 http(s) 环境：

```bash
cd image-studio
python3 -m http.server 8080   # 然后打开 http://localhost:8080
# 或：npx serve image-studio
```

或部署到 GitHub Pages，访问仓库的 `/image-studio/` 路径。

**接入真实模型**：右上角 ⚙ → 选 OpenAI / 自定义 → 填 Base URL、API Key、图像模型（`gpt-image-1`）、文本模型（`gpt-4o-mini`）→ 保存。

> - 浏览器直连 OpenAI 受 **CORS** 限制；官方端点通常允许，自建端点请放开跨域。
> - `gpt-image-1` 部分账号需先完成**组织验证**才能调用；也可在设置里把图像模型改成 `dall-e-3`（代码会自动适配 `n=1`、尺寸与 `response_format` 等差异）。

## 🎨 设计思路（面试讲解要点）

- **工作流的价值在于把「提示词」显式化**。普通图像产品把这步藏进黑盒；这里刻意把它拆成第一道可见、可编辑的工序，因为 AIPM 的判断之一就是**知道一条链路里真正决定质量的环节在哪、该不该把控制权留给人**。
- **为什么不做成 Agent**。出图本质是单步动作，硬套 ReAct 循环只是表演。这里诚实地做成**确定性工作流**——可预测、便宜、对「多张抽选」体验更友好；与同仓库 PRD Agent（多步自治）刚好互为对照。
- **零配置可跑 + BYOK 真实驱动**，共用同一契约：演示现场网络/Key 不可控时也能稳定展示，同时不牺牲「能接真模型」的可信度。
- **跨模型适配**：图像模型在 `gpt-image-1` / `dall-e-3` 之间切换时，参数差异（n、尺寸、quality、response_format）由 `openai.js` 统一收敛，对上层透明。

## 🗺 后续可迭代方向

- 提示词「自动调优」**导演模式**：出图后用 vision 自评、改提示词、重生，直到达标或到上限（真正的 agentic loop）。
- 历史记录支持保存完整原图（IndexedDB）、按标签检索与一键复跑。
- 局部重绘（mask）、风格参考图、批量出图与下载。
- 出图参数预设（海报 / 图标 / 头像）一键带入比例与质量。
